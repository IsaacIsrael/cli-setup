# Concepts

`cli-setup` is built around a small, consistent vocabulary — a shared set of terms
(a *tool*, a *profile*, a *team config*, and so on) that recur throughout the
CLI's output and the rest of this documentation. Naming these ideas once keeps the
behavior predictable and the docs unambiguous.

This section grows with the CLI: each concept gets documented as the feature that
introduces it ships. Until then, the [roadmap](roadmap.md) describes what is
planned.
